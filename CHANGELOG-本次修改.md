# 本次修改总结

## 阶段一：素材导入模块
- **拖拽逻辑**：从拖入的文件路径中提取**父目录**作为文件夹路径，支持多文件夹；Toast 显示新增文件夹数量。
- **点击浏览**：Dropzone 支持点击打开文件夹选择；清空列表时同时清空尺寸勾选与项目名。

## 阶段二：校验与目录结构
- **主进程**：`fs:startValidation` 改为**深度递归**扫描（`fs.stat` 判断目录），支持中文子文件夹；返回字段增加 `folderName`，尺寸子文件夹命名改为**纯数字**（如 `1080x1920`，无中文后缀）。
- **表格展示**：表格列展示文件名（含扩展名）、文件夹名、格式（大写无点）、尺寸、状态；缺失行显示目标尺寸。
- **批量初始化目录**：`fs:initFolders` 改为接收 `projectsData[]`，弹窗选择目标总目录后批量创建多项目；子文件夹仅用数字+小写 x（如 `720x1280`），每尺寸下建 `_Assets`。
- **JSON 解析**：`dialog:openJson` 支持孟祥伟数据表数组格式，返回 `projects` 数组供批量建目录；**导入 JSON 不再自动修改左侧尺寸勾选**，仅用于一键初始化目录。

## 阶段一增强：自动识别尺寸并勾选
- **主进程**：新增 `fs:readProjectSizes(folderPaths)`，读取各路径下**一级子目录**名，识别 `720x1280` 等尺寸格式，去重后返回规范化尺寸数组（`*` 分隔）。
- **渲染进程**：添加文件夹（按钮或拖拽）后调用 `readProjectSizes`，将识别到的尺寸自动写入 `selectedSizes`，左侧尺寸胶囊自动勾选，便于直接校验/重命名。

## 验收要点
- 导入 JSON → 一键初始化目录 → 本地生成多项目+纯数字尺寸子文件夹。
- 拖入/添加项目文件夹 → 左侧尺寸自动勾选 → 开始校验/执行重命名流程顺畅。
